# 飞书多维表格 AI 提示词合集（可直接粘贴）

> 用法:进入多维表格 → 用「AI 建表 / 智能创建」粘贴 A 组提示词生成表结构;
> AI 字段用 B 组;仪表盘用 C 组;智能体用 D 组。
> 提示:结构用 AI 生成后,数据可直接导入本目录的 4 个 CSV（已含"数据侦探"完整故事线）。

---

## A. 建表提示词

### A0 · 一键生成 4 张表（推荐,粘到「AI 建表」）

```
帮我创建一个用于「制造业千人千面数据洞察」的多维表格，包含以下 4 张数据表，请按字段名和类型建好：

1. 表名「全球生产运营总表」，字段：记录ID(文本)、工厂(单选:潍坊/越南/青岛)、产线(单选:L1/L2/L3/L4)、时间戳(日期时间)、OEE(数字,百分比)、FPY(数字,百分比)、UPH(数字)、设备状态(单选:运行/停机)、异常代码(文本)、数据上报时间(日期时间)。

2. 表名「异常记录表」，字段：异常ID(文本)、关联记录ID(文本)、工厂(单选)、产线(单选)、异常描述(多行文本)、发生时间(日期时间)、分类(单选:设备故障/来料问题/工艺偏差/数据质量/人员操作)、责任人(人员)、故障代码(文本)、处置建议(多行文本)。

3. 表名「来料批次表」，字段：批次号(文本)、供应商(文本)、物料(文本)、来料日期(日期)、质检结果(文本)、不良率(数字,百分比)、关联产线(文本)。

4. 表名「预警任务表」，字段：任务ID(文本)、触发时间(日期时间)、来源记录ID(文本)、异常摘要(多行文本)、严重级别(单选:严重/一般)、责任人(人员)、状态(单选:待处理/处理中/已处理)、处理措施(多行文本)、处理时间(日期时间)、复盘结果(多行文本)。
```

## B. AI 字段捷径提示词（在「异常记录表」新增字段 → 字段捷径 / AI 能力）

**B1 · AI 自动分类**（字段名:分类）
- 数据来源字段:异常描述
- 分类标签:设备故障 / 来料问题 / 工艺偏差 / 数据质量 / 人员操作
- 提示词:
```
阅读「异常描述」，将该异常归入以下类别之一：设备故障、来料问题、工艺偏差、数据质量、人员操作。只输出类别名称。
```

**B2 · 信息提取**（字段名:故障代码）
```
从「异常描述」中提取故障或报警代码（如 E-207 这类编号），如果没有则输出：无。
```

**B3 · AI 智能总结**（字段名:处置建议）
```
阅读「异常描述」，用一句话给出可执行的处置建议，30 字以内，先动作后原因。例如：立即停机检修主轴冷却系统，因温度超限。
```

## C. 仪表盘生成提示词（点「仪表盘」→ 多维表格 Agent / AI 生成）

**C1 · 生产岗**
```
基于「全球生产运营总表」生成生产岗看板：各工厂 OEE 对比柱状图、各产线 UPH 排名、今日达成率指标卡、异常事件列表。
```

**C2 · 质量岗**
```
基于「全球生产运营总表」和「异常记录表」生成质量岗看板：各产线 FPY 趋势折线图、按「分类」字段的缺陷分布饼图、FPY 低于 93% 的产线预警列表。
```

**C3 · 设备岗**
```
基于「全球生产运营总表」生成设备岗看板：设备 OEE 排行、状态为「停机」或有异常代码的设备列表、故障响应时效统计。
```

**C4 · 管理岗（KPI 驾驶舱）**
```
基于全部数据表生成管理岗 KPI 驾驶舱：全球工厂 OEE/FPY/UPH 汇总指标卡、跨工厂对比图、严重级别为「严重」的预警任务汇总。
```

**C5 · 动态角色感知（同一岗位按业务阶段切视图）**
```
在质量岗看板里再建一个视图「交付冲刺期」，筛选出 FPY 低于 93% 且 UPH 低于目标的瓶颈产线，突出显示齐套/来料风险。
```

## D. 多维表格智能体设置（「创建智能体」）

- 名称:生产数据侦探
- 关联数据表:全球生产运营总表、异常记录表、来料批次表、预警任务表
- 指令 / 人设:
```
你是制造数据分析助手「生产数据侦探」。基于关联的四张数据表回答用户问题，规则：
1. 先给结论，再给数据依据（定位到具体工厂/产线/时间）。
2. 遇到良率(FPY)或 OEE 异常，主动跨表关联「来料批次表」和「异常记录表」做归因，指出可能根因。
3. 涉及处置进展时，查「预警任务表」给出责任人、措施与复盘结果。
4. 回答简洁，不编造数据表中没有的内容。
```
- 测试问题:
  - 为什么 07/15 潍坊 L2 的 FPY 下降了？
  - 现在哪些产线 FPY 低于 93%？
  - 潍坊 L2 的问题处理了吗，效果如何？

## 演示故事线（数据已在 CSV 中埋好）

潍坊 L2 的 FPY 在 07/15 骤降至 91% → 跨表归因锁定来料批次 B20260714-03 电容容值超差 → 隔离批次并整改 → 07/17 FPY 回升至 94.6%，AI 复盘确认闭环有效。六类数据缺陷分散在 R005–R012 各行，用于演示「数据前置体检」。
